Privacy Policy

Privacy

Dayside is a local, offline tool. This document describes exactly what data it accesses and what it does with it.

🌐
No network calls

Dayside never connects to the internet. No telemetry, no analytics, no update checks.

🎙️
No recording

Screen, camera, and microphone are never accessed.

💾
No persistent data

Nothing is written to disk. When the app closes, nothing remains.

🔒
No biometrics

No face scan, voiceprint, or keystroke logging of any kind.

✋
No process termination

Dayside is strictly read-only. It never kills, suspends, or modifies any process.

✅
Explicit consent

No scan runs unless the user explicitly clicks "Start scan" after reading the consent screen.

What Dayside reads during a scan

  • Running processes — names and paths, matched against a blocklist
  • Open windows — titles and screen-capture exclusion flags
  • Audio/video devices — device names, to detect virtual cameras and audio cables
  • Browser tab titles — on Windows; titles and URLs on macOS
  • System state — monitor count, RDP/screen-sharing session status

All of this is processed locally on the candidate's machine and never transmitted anywhere.

Consent

Every session begins with a consent screen that explains what the scan will do. No scan runs unless the user explicitly clicks "Start scan" after reading and accepting. Periodic re-scanning is off by default and requires the user to opt in.

Data shared with the interviewer

The only data shared is what appears on the candidate's screen during their screen-share session — the scan results panel that both parties look at together. No data is transmitted electronically to the interviewer or to any third party.

Open source

Dayside is fully open source under the Apache License 2.0. Anyone can read the code, build it from source, and verify these claims independently. There is no hidden logic, no obfuscated binary, and no server-side component.
