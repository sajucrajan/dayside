Dayside is a candidate-operated pre-interview machine check. It flags known AI interview assistants and stealth tools — on the candidate's own screen, in plain sight of the interviewer.

⬇ Download latest View on GitHub

How it works

Six steps. The candidate is in control at every one.

1
Candidate downloads and runs Dayside

A consent screen appears explaining exactly what the tool will do. The candidate agrees or declines — declining closes the app immediately.

2
Main window opens in "Ready" state

Nothing scans yet. The candidate clicks Start scan when they're ready — typically at the start of the screen-share.

3
One scan runs (~10 seconds)

Results appear on the candidate's screen. The candidate and interviewer discuss anything flagged together.

4
Optional: "Keep checking during interview"

A checkbox that, if ticked by the candidate, re-runs the scan every 5 minutes. Off by default. The candidate chooses.

5
Compact pill mode

The window can shrink to a small always-on-top status pill that stays visible during the interview, showing scan status and last scan time.

6
Interview ends — candidate closes the app

Nothing persists. No data leaves the machine. Ever.

What it checks

All checks run locally. Nothing is uploaded.

⚙️
Running processes

Names and signatures matched against a maintained blocklist of known AI interview assistants and stealth tools.

🪟
Visible & cloaked windows

Including windows that set themselves "excluded from screen capture" — the invisible overlay pattern.

🎥
Virtual audio & video devices

Virtual cameras (deepfake vectors) and virtual audio cables (voice-routing vectors).

🌐
Remote-access & tunneling tools

TeamViewer, AnyDesk, Parsec, RustDesk, ngrok, cloudflared, Tailscale, and more.

🗂️
Browser tabs

Titles on Windows, titles plus URLs on macOS — checked against known AI-assistant domains and LLM chat sites.

🖥️
System state

Monitor count, RDP / screen-sharing session status, architecture.

Privacy by design

Dayside is read-only and local-only.

✓No network calls — ever
✓No recording of screen, camera, or mic
✓No biometrics of any kind
✓No persistent data — closes clean
✓No process termination
✓Explicit consent before every scan
✓Fully open source — verify it yourself
✓Apache 2.0 license

For employers

Guidance on responsible deployment.

Dayside is a conversation tool, not a verdict machine. Every flag is a question to ask, not a reason to disqualify. Candidates must receive advance notice, refusal must be a real option, and an alternative must be offered to those who cannot or will not run the tool.
